# ADR-014: Domain Type Quantifiability — Two Orthogonal Validation Checks

**Status:** Accepted  
**Date:** 2026-03-28  
**Tags:** type-safety, domain-types, query-binding, model-validation, quantifiability

---

## Context

- A `TypeCatalog` partitions its declared types into **domain types** and **value types** via tagged declarations (`DomainType(id)` / `ValueType(id)`) — domain types are first-class entities that can be quantified over (`∀x:Asset`); value types are scalars that flow through function/predicate arguments but are never iterated
- A query that quantifies over a value type is a **query-language misuse**: the violation is fully detectable at bind time from the catalog alone, independent of any `RuntimeModel`
- A `RuntimeModel` that omits element sets for one or more declared domain types is an **infrastructure misconfiguration**: the violation is detectable at startup by checking the model against the catalog, independent of any specific query
- These two failures are **orthogonal** — the bind-time check operates on `(ParsedQuery, TypeCatalog)`; the startup check operates on `(RuntimeModel, TypeCatalog)`. Neither subsumes the other
- Conflating the two, or dropping the bind-time check on the assumption that the startup check "covers it", causes query-language misuse to reach the evaluator and surface as `DomainNotFoundError` — a defensive fallback that is only reachable by bypassing the intended pipeline

---

## Decision

### 1. `TypeDecl` tags encode quantifiability at construction

Each type is declared as either `DomainType` (quantifiable) or `ValueType` (not
quantifiable) inline in the `types` set. There is no separate `domainTypes`
parameter; `catalog.domainTypes` is a derived method that collects the
`DomainType` ids. This makes mismatches structurally impossible:

```scala
TypeCatalog.unsafe(
  types = Set(
    DomainType(asset), DomainType(risk),     // ∀x:Asset and ∀x:Risk are legal
    ValueType(loss), ValueType(probability)  // ∀x:Loss and ∀x:Probability are TypeNotQuantifiable
  ),
  predicates = Map(
    SymbolName("has_risk") -> PredicateSig(List(asset, risk)),
    SymbolName("hasloss")  -> PredicateSig(List(loss))        // Loss OK as param, not as var
  ),
  functions = Map(
    SymbolName("loss_of") -> FunctionSig(List(risk), loss)    // returns Loss — fine
  )
)

### 2. `QueryBinder.bind` rejects quantification over value types

When the root variable or an inner `∀`/`∃` variable resolves to a type not in
`domainTypes`, `bind` returns `TypeCheckError.TypeNotQuantifiable`. This check
requires only the catalog — no runtime model required:

```scala
// catalog.domainTypes = Set(asset, risk); Loss is a value type
val catalog = TypeCatalog.unsafe(
  types = Set(DomainType(asset), DomainType(risk), ValueType(loss)),
  predicates = Map(SymbolName("hasloss") -> PredicateSig(List(loss)))
)
val query = ParsedQuery(
  quantifier = Quantifier.allOf,
  variable   = "l",
  range      = FOL("hasloss", List(Term.Var("l"))),  // l inferred as TypeId("Loss")
  scope      = Formula.True
)
QueryBinder.bind(query, catalog)
// Left(List(TypeCheckError.TypeNotQuantifiable("Loss")))
```

### 3. `RuntimeModel.validateAgainst` enforces domain coverage at startup

A model is misconfigured if any `domainTypes` entry has no corresponding element
set in `domains`. This check fires at application startup — before any query is
served — independently of any query:

```scala
// Catalog declares both Asset and Risk as domain types
val catalog = TypeCatalog.unsafe(
  types = Set(DomainType(asset), DomainType(risk))  // both must have element sets
)

// Model provides elements only for Asset — Risk element set omitted
val model = RuntimeModel(
  domains    = Map(asset -> Set(vA, vB)),   // risk absent ← misconfiguration
  dispatcher = dispatcher
)

model.validateAgainst(catalog)
// Left(List(RuntimeModelError.MissingDomainForType(TypeId("Risk"))))
// → application must NOT begin serving queries
```

## Code Smells

### ❌ Dropping the QueryBinder check because `validateAgainst` "covers it"

```scala
// BAD: without the bind-time check, Loss quantification reaches the evaluator
QueryBinder.bind(query, catalog)   // succeeds — variable "l" is type Loss
TypedSemantics.evaluate(bound, model)
// → DomainNotFoundError("Loss") at evaluation time  ← wrong error, wrong time

// GOOD: bind-time check catches it before any model is involved
QueryBinder.bind(query, catalog)
// Left(List(TypeCheckError.TypeNotQuantifiable("Loss")))
```

### ❌ Using `DomainNotFoundError` to signal query-language misuse

```scala
// BAD: treating a structural query error as a missing-data error
case Left(e: QueryError.DomainNotFoundError) =>
  // retry with a different model?  ← wrong — the query itself is invalid

// GOOD: distinct error types, unambiguous handling
case Left(e: QueryError.BindError) if e.errors.exists(_.contains("TypeNotQuantifiable")) =>
  // reject query: Loss is a value type, cannot quantify over it
case Left(e: QueryError.ModelValidationError) =>
  // fatal: model is missing Risk element set — do not serve queries
case Left(e: QueryError.DomainNotFoundError) =>
  // defensive fallback — should not be reachable in a correctly wired system
```

### ❌ Encoding quantifiability solely via presence in `RuntimeModel.domains`

```scala
// BAD: "if it has a domain it's quantifiable" — requires a model to rule on catalog intent
def isQuantifiable(t: TypeId, model: RuntimeModel): Boolean =
  model.domains.contains(t)    // wrong: this is a model fact, not a schema fact

// GOOD: quantifiability is declared in the catalog, model provides the elements
def isQuantifiable(t: TypeId, catalog: TypeCatalog): Boolean =
  catalog.domainTypes.contains(t)
```

## Cross-ADR Relationship

ADR-001 covers the full typed-pipeline compilation flow and documents
`validateAgainst` as the startup infrastructure check (§4). This ADR clarifies
that `validateAgainst` is **one** of two required validations. ADR-001 §2
(`QueryBinder.bind` as the single compilation step) is where the
`TypeNotQuantifiable` check lives — this ADR specifies its scope and the
reason it must not be removed.

## Implementation

|---|---|
| `fol/typed/TypeDefs.scala` | `sealed trait TypeDecl`; `DomainType(id: TypeId)` and `ValueType(id: TypeId)` sub-cases |
| `fol/typed/TypeCatalog.scala` | `types: Set[TypeDecl]` field; `domainTypes` derived via `.collect { case DomainType(id) => id }`; no `domainTypes` construction parameter |
| `fol/typed/TypeCheckError.scala` | `TypeNotQuantifiable(name: String)` bind-time error case |
| `fol/typed/QueryBinder.scala` | Check root variable and nested quantifiers against `catalog.domainTypes` |
| `fol/typed/RuntimeModelError.scala` | `MissingDomainForType(typeName: TypeId)` startup error case |
| `fol/typed/RuntimeModel.scala` | `validateAgainst` checks `domainTypes.diff(domains.keySet)` |
| `fol/semantics/VagueSemantics.scala` | `renderTypeErrors` arm for `TypeNotQuantifiable`; `renderModelErrors` arm for `MissingDomainForType` |
| `fol/typed/QueryBinderSpec.scala` | Bind-time rejection tests for value-type quantification |
| `fol/semantics/VagueSemanticsTypedSpec.scala` | Startup rejection tests for missing domain element sets |
