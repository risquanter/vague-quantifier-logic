# ADR-003: HDR Deterministic Sampling

**Status:** Accepted  
**Date:** 2026-03-21  
**Tags:** sampling, PRNG, HDR, Fisher-Yates, reproducibility

---

## Context

- Vague quantifier evaluation on large domains requires sampling a subset of elements rather than checking every element — the sample must be statistically representative and yield valid confidence intervals
- Standard `java.util.Random` / `scala.util.Random` produce different sequences across platforms, JVM versions, and languages — making cross-platform reproducibility impossible
- Monte Carlo simulations in the risk register need deterministic, reproducible random sequences that can be correlated across entities and variables without shared mutable state
- A counter-based PRNG allows direct access to any trial `N` without computing trials `0..N-1` — enabling parallel evaluation and index-based sampling patterns

---

## Decision

### 1. HDR Counter-Based PRNG

All randomness flows through the HDR (Hubbard Decision Research) PRNG.
No `scala.util.Random`, no `java.util.Random` anywhere in the codebase.
HDR uses a 4-layer seed hierarchy for deterministic, multi-dimensional sequences:

```scala
// fol.sampling.SamplingParams
case class HDRConfig(
  entityId: Long = 0L,   // Layer 1: organization/domain
  varId:    Long = 0L,   // Layer 2: relation/variable
  seed3:    Long = 0L,   // Layer 3: scenario
  seed4:    Long = 0L    // Layer 4: trial offset
)
```

### 2. Fisher-Yates Shuffle for Sample Selection

`HDRSampler` uses partial Fisher-Yates shuffle — exactly `n` HDR trials
to select `n` elements, deterministic performance, no collision handling:

```scala
// fol.sampling.HDRSampler
class HDRSampler[A](config: HDRConfig)(using ClassTag[A]) extends Sampler[A]:
  def sample(population: Set[A], n: Int): Set[A] =
    val elements = population.toArray
    val N = elements.length
    val actual = math.min(n, N)
    // Partial Fisher-Yates: shuffle first `actual` positions
    for i <- 0 until actual do
      val randomValue = hdr.trial(i.toLong)
      val j = i + (randomValue * (N - i)).toInt
      val temp = elements(i)
      elements(i) = elements(j)
      elements(j) = temp
    elements.take(actual).toSet
```

### 3. `SamplingParams` Controls Precision

`SamplingParams(epsilon, alpha)` determines sample size via the sample
size formula with finite population correction. `SamplingParams.exact`
(ε = 1e-6) forces n = N — "exact" is the degenerate case of "sampled":

```scala
case class SamplingParams(epsilon: Double = 0.1, alpha: Double = 0.05)

object SamplingParams:
  val default:      SamplingParams = SamplingParams()             // 10% error, 95% CI
  val conservative: SamplingParams = SamplingParams(0.05, 0.01)   // 5% error, 99% CI
  val exact:        SamplingParams = SamplingParams(1e-6, 0.05)   // forces n = N
```

### 4. Wilson Score Confidence Interval

`ProportionEstimator` computes Wilson score CI with finite population
correction (FPC). Pure-Scala inverse normal (Acklam + Abramowitz-Stegun
refinement) — no `commons-math3` dependency:

```scala
// fol.sampling.ProportionEstimator
def estimateWithSampling(
  elements: Set[RelationValue],
  predicate: RelationValue => Boolean,
  params: SamplingParams,
  hdrConfig: HDRConfig,
  populationSize: Int
): ProportionEstimate
// Returns: ProportionEstimate(p, n, k, ciLower, ciUpper)
```

## Code Smells

### ❌ Using `scala.util.Random`

```scala
// BAD: Non-reproducible, platform-dependent
val rng = new scala.util.Random(seed)
val sample = rng.shuffle(population.toList).take(n).toSet

// GOOD: HDR deterministic sampling
val sampler = HDRSampler[RelationValue](hdrConfig)
val sample = sampler.sample(population, n)
```

### ❌ Boolean Toggle for Exact vs Sampled

```scala
// BAD: Two code paths
def evaluate(useSampling: Boolean): VagueQueryResult =
  if useSampling then evaluateWithSampling() else evaluateExact()

// GOOD: SamplingParams controls mode — one code path
val params = if exactMode then SamplingParams.exact else SamplingParams.default
resolved.evaluate()  // params sealed in ResolvedQuery
```

### ❌ External Dependency for Inverse Normal

```scala
// BAD: Pull in commons-math3 for one function
import org.apache.commons.math3.distribution.NormalDistribution
new NormalDistribution().inverseCumulativeProbability(p)

// GOOD: Pure-Scala Acklam approximation (max error < 1.15e-9)
NormalApprox.inverseNormal(p)
```

## Implementation

| Location | Pattern |
|---|---|
| `fol/sampling/HDRSampler.scala` | Fisher-Yates shuffle with HDR PRNG |
| `fol/sampling/SamplingParams.scala` | `SamplingParams` + `HDRConfig` types |
| `fol/sampling/SampleSizeCalculator.scala` | Cochran formula + FPC → sample size `n` |
| `fol/sampling/ProportionEstimator.scala` | Single-pass filter + count + Wilson CI |
| `fol/sampling/NormalApprox.scala` | Pure-Scala inverse normal (Acklam + A&S) |
| `fol/query/ResolvedQuery.evaluate()` | Calls `ProportionEstimator` with sealed params |
