# ADR-002: Parser-Combinator Style

**Status:** Accepted  
**Date:** 2026-03-21  
**Tags:** parser, CPS, error-handling, OCaml, combinators

---

## Context

- Parser combinators must compose: a quantifier parser, a variable parser, and a formula parser combine without glue code or intermediate error types
- OCaml's `try/with Failure` pattern gives backtracking for free — each combinator either returns `(result, remaining_tokens)` or throws, and the caller decides whether to retry
- Wrapping every combinator in `Either` forces for-comprehension chains that obscure the grammar and duplicate error-handling at each layer
- The FOL parser core (`Combinators`, `FormulaParser`, `FOLAtomParser`, `Lexer`) uses `List[String] => (A, List[String])` — the vague layer must extend this, not reinvent it

---

## Decision

### 1. `ParseResult[A]` as the Universal Return Type

Every parser function — FOL or vague — returns `(A, List[String])`.
No `Either`, no `Option`, no custom monad inside the parser:

```scala
// parser.Combinators
type ParseResult[A] = (A, List[String])

// fol.parser.VagueQueryParser — same pattern
def parseTokens(tokens: List[String]): (ParsedQuery, List[String]) =
  val (quantifier, t1) = parseQuantifier(tokens)
  val (variable, t2)   = parseVariable(t1)
  val t3               = expect("(", t2)
  val (range, t4)      = FOLAtomParser.parseAtom(List(), t3)
  val t5               = expect(",", t4)
  val (scope, t6)      = FormulaParser.parse(...)(t5)
  val t7               = expect(")", t6)
  val (answerVars, t8) = parseAnswerVars(t7)
  (ParsedQuery.mk(quantifier, variable, range, scope, answerVars), t8)
```

### 2. Throw on Error, Catch at the Edge

Internal parsers throw `QueryException`. A single `try/catch` at the
public entry point converts to `Either[QueryError, A]`:

```scala
// Public API — one boundary
def parse(s: String): Either[QueryError, ParsedQuery] =
  try
    val tokens = Lexer.lex(explode(s))
    val (query, remaining) = parseTokens(tokens)
    if remaining.nonEmpty then Left(QueryError.ParseError(...))
    else Right(query)
  catch case e: QueryException => Left(e.toError)

// Internal — throws, no Either
private def parseQuantifier(tokens: List[String]): (Quantifier, List[String]) =
  tokens match
    case "Q" :: "[" :: op :: "]" :: "^" :: "{" :: rest => ...
    case _ => throw QueryException(QueryError.ParseError(...))
```

### 3. Composition via Token Threading

Combinators compose by passing the remaining token list forward.
Generic infrastructure (`parseLeftInfix`, `parseRightInfix`,
`parseBracketed`, `parseList`) handles precedence and grouping:

```scala
// parser.Combinators — generic infix parsing
def parseRightInfix[A](
  opSym: String, opCons: (A, A) => A,
  subParser: List[String] => ParseResult[A]
)(inp: List[String]): ParseResult[A] =
  val (left, rest) = subParser(inp)
  if nextin(rest, opSym) then
    val (right, rest2) = parseRightInfix(opSym, opCons, subParser)(rest.tail)
    (opCons(left, right), rest2)
  else (left, rest)
```

### 4. Lexer Produces Flat Token List

`Lexer.lex` tokenizes the input string into `List[String]`.
All parsers operate on this flat list — no intermediate AST,
no token objects, no position tracking beyond the list itself:

```scala
// lexer.Lexer
def lex(input: List[Char]): List[String]

// Tokens are plain strings: "Q", "[", ">=", "]", "^", "{", "3", "/", "4", ...
```

## Code Smells

### ❌ Either Inside Combinators

```scala
// BAD: Each combinator returns Either, requiring for-comprehension
def parseTokensInternal(tokens: List[String]): Either[QueryError, (ParsedQuery, List[String])] =
  for
    (quantifier, t1) <- parseQuantifierInternal(tokens)
    (variable, t2)   <- parseVariableInternal(t1)
    ...
  yield (query, remaining)

// GOOD: Combinators return tuples, throw on failure
def parseTokens(tokens: List[String]): (ParsedQuery, List[String]) =
  val (quantifier, t1) = parseQuantifier(tokens)
  val (variable, t2)   = parseVariable(t1)
  ...
  (query, remaining)
```

### ❌ Wrapper Pairs (*Internal + throwing)

```scala
// BAD: Two methods for the same operation
def parseQuantifierInternal(tokens: List[String]): Either[QueryError, ...]
def parseQuantifier(tokens: List[String]): (Quantifier, List[String]) =
  parseQuantifierInternal(tokens).fold(e => throw ..., identity)

// GOOD: One method, one style
def parseQuantifier(tokens: List[String]): (Quantifier, List[String]) =
  tokens match
    case "Q" :: ... => ...
    case _ => throw QueryException(...)
```

### ❌ Re-wrapping Exceptions Mid-Flight

```scala
// BAD: Catch exception from layer below, wrap in Either, then caller unwraps
def parseFormulaSafe(tokens: List[String]): Either[QueryError, ...] =
  try Right(FormulaParser.parse(...)(tokens))
  catch case e: Exception => Left(QueryError.ParseError(e.getMessage, ...))

// GOOD: Let exceptions propagate, catch once at the edge
val (scope, t6) = FormulaParser.parse(...)(t5)  // throws propagate naturally
```

## Implementation

| Location | Pattern |
|---|---|
| `parser/Combinators.scala` | `ParseResult[A]` type, generic infix/bracket/list combinators |
| `parser/FormulaParser.scala` | FOL formula parsing via combinator composition |
| `parser/FOLAtomParser.scala` | Atom/term parsing — throws on malformed input |
| `lexer/Lexer.scala` | `String → List[String]` tokenization |
| `fol/parser/VagueQueryParser.parse()` | Single `try/catch → Either` boundary |
| `fol/parser/VagueQueryParser.parseTokens()` | Vague query combinator — extends FOL parsers |
